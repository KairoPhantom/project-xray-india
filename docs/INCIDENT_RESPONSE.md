# Incident response

1. Triage severity and appoint incident commander.
2. Preserve logs/evidence; do not destroy disputed material.
3. Contain credentials, affected endpoints or publication.
4. Notify affected people/partners when required.
5. Correct public record with transparent version note.
6. Restore verified service and monitor.
7. Publish post-incident review for material trust failures.

Severity 0 includes whistleblower exposure, fabricated evidence publication, unauthorized claim publication or audit-log compromise.
