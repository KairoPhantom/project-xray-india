# Product requirements document

## Problem
Indian public-infrastructure evidence is fragmented across procurement portals, authority sites, PDFs, audit reports, news, RTI responses and field observations. Users cannot easily distinguish verified facts, allegations, official explanations, missing records and contractual obligations.

## Primary users
1. Investigative journalist assembling a reliable project timeline.
2. Civil engineer evaluating whether public claims match accessible technical evidence.
3. Civic researcher generating precise information requests.
4. Auditor/funder preparing an implementation review.
5. Authority or contractor submitting a source-backed response/correction.

## Core jobs
- Show what was promised, changed, certified, paid and reported.
- Preserve source provenance.
- Identify records not located in a declared search scope.
- Produce review questions and RTI drafts.
- Maintain corrections and right-to-respond.

## P0 functional requirements
- Project dossier; source/claim/document/gap/response management; audit log; exports; reviewer gate; authentication; mobile public view; correction route; backup/restore.

## Non-goals
- Guilt prediction, autonomous investigations, actual payment, anonymous leaks, nationwide crawling, financial advice, engineering certification, legal advice.

## Success metrics
- Median verified dossier assembly time.
- Percentage of published claims with exact passage/page.
- Corrections per 100 claims.
- Missing records later disclosed.
- Qualified journalist/engineer/CSO usage.
- Authority responses and resolved issues.
- Zero unsupported allegations published.
