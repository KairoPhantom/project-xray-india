# Contributing

Use signed commits where possible. Open an issue before changing evidence states or publication policy. All code changes require tests. Case-data changes require source review by two maintainers who did not author the extraction. Never submit personal data, secrets, unredacted whistleblower material or unsupported allegations.

## Commit convention
`feat:`, `fix:`, `docs:`, `test:`, `security:`, `data:`.

## Case-data pull requests
Include original source, archived copy where lawful, retrieval timestamp, exact passage/page, claim classification, reviewer names and correction path.
